1 git-tutor.pdf

1.1 一般使用

安装好后, 可以先设置下用户名和密码, 尤其协作时比较需要, 设置一次就可以了.

git config --global user.name "Your Name Comes Here"
git config --global user.email you@yourdomain.example.com

简单三步

git init
git add filelist 
#(对应内部命令: git-update-index --add, 还可以 git-update-index --remove)
git commit -m "msg"

索引更新较方便的方法

git add . 目录下所有文件
例外文件 > .gitignore

以后更新如果索引不变, 则直接

git commit -a -m "msg"

1.2 reset

  • mixed: 仅重置索引
  • soft: 索引, 文件都不变, 但已经提交变为未提交. 用于更改当前提交, 不会产生无用提
    交. 典型用法:
git-reset --soft HEAD^
#修改文件
git-commit -a -c ORIG_HEAD

1.3 branch

创建 git branch new 创建的分支是原当前分支的 copy
转到 git checkout new
创建并转到 git checkout -b new
删除 git branch -D new
查看 git branch
更细查看 git show branch

1.4 其它

git-merge "Merge work in robin" HEAD robin
git-pull . robin
git-show-branch
git-checkout -f foo.c

2 廖雪峰的 git

command comment
git status 工作区的状态
git diff 查看修改内容
git diff HEAD -- readme.txt 查看工作区和版本库里面最新版本的区别
git log --pretty=oneline --abbrev-commit 找到历史提交的 commit id
git reflog 查看命令历史
git reset --hard commit_id 在版本的历史之间穿梭
git checkout -- readme.txt 丢弃工作区的修改
git reset HEAD file 把暂存区的修改撤销掉（unstage）
git checkout -b dev 创建并切换到 dev 分支
git merge dev 合并 dev 分支到当前分支
git branch -d <name> 删除分支
git tag -a <name> -m <comment> <commitID> 指定标签信息 
#beautiful git log with color
git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

2.1 git 技巧

  • 内建的图形化 git：gitk
  • 彩色的 git 输出：git config color.ui true
  • 显示历史记录时，每个提交的信息只显示一行：git config format.pretty oneline
  • 交互式添加文件到暂存区：git add -i

3 pro_git

3.1 cmd

## 找出分支引用的 commit 对象 SHA1 值
git rev-parse 分支名/HEAD
## 查看引用日志
git reflog
## 引用的写法
git show master
develop@{yesterday/2.months.ago}
HEAD@{5}
HEAD^^
HEAD~3
master..experiment #在 experiment 不在 master 中
experiment ^master/--not master #同上
--left-right master...experiment #分别在 experiment 或 master 中, 但非其共同有的 commit 对象
