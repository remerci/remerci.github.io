1 emacs 预览支持

两种方法：

  1. 来自Ivaldi，将meta-mode.el和pi-meta.el放到 emacs 的 load path，将pi-anime-meta.sh放到~/bin 目录。在.emacs 文件中加入配置：

<src lang="elisp">
;; ————
;; * Metapost *
(load "pi-meta")
</src>

使用：

  • C-c C-c 编译预览
  • C-c C-p 用 beamer 生成动画
  • M-x customize group <RET> pi-meta <RET> 配置 pi-meta
  • 最新方案，在 mp 文件中加入下列代码，用 mpost 编译。可将字体嵌入，并可将名字后缀加入 eps。

<example>
prologues:=3;
filenametemplate "%j-%c.eps";
</example>

来自ctex，将 perl 脚本my_metapost放到~/bin 目录。.emacs 中如下配置：
<src lang="elisp">
(defun mpost-compile () (interactive)
(save-buffer)
(shell-command (format mpost-compile-command buffer-file-name))
)
(defvar mpost-compile-command "my_metapost %s")
(add-hook 'metapost-mode-hook
'(lambda nil
(define-key meta-mode-map "\C-c\C-c" 'mpost-compile)
)
)
</src>

该脚本的大概原理是在文件头自动加上 prologues:=3; 在用 mpost 编译 filename.mp 后就会生成 filename.1，filename.2，等 eps 文件，把这些文件依次改名为 filename-1.eps，filename-2.eps，再自动调用 eps 的阅读器一个一个地预览（在脚本中要把预览改为 yes，再把浏览器改成了 gv 等等）。

总的来说，pi-meta.el 和 pi-anime-meta.sh 的功能齐全，扩展了原来的 metapost-mode 编辑功能（如自动补全()，[]，""，{}等)，完全不依赖 perl 和其他脚本，无须用到 metapost 的新版本中的 prologues:=3;的功能，也没有以前的一些限制，特别是对于那些 beginfig(*);*处并非标准数字（比如为了生成一系列图形调用 for 循环用了 beginfig{i+10}等不太标准的形式），仍然可以正确生成预览（关于这方面请参见 /usr/local/texlive/2007/texmf-dist/doc/metapost/mp3d/examples/目录下的例子，还有 Anthony Phan 的 m3D 宏包中的例子等等）。但它们的不足之处是预览比起 instanton 写的 perl 脚本要慢，因为多了 latex，dvips 或 latex,dvips,ps2pdf 的过程，有鉴于此，我们可以改动 pi-meta.el 的第 202 和 203 行，比如我们可以把 203 行的[(control c) (control c)]第二个 c 改为 v，则以后 C-c C-v 才调用 pi-meta.el 预览，平常还是用 C-c C-c 调用 instanton 的 perl 脚本编译和预览。

3 画图技巧

3.1 用 beamer 作动画

用 mutliinclude 可以将一个 mp 文件中的多个 fig 用多个页面逐步显示。第一页显示 fig0，第二页显示 fig0 和 fig1，以此类推。metapost 文件中 fig0 里定义的点间关系后面的 fig 也可以用，因此将一个 fig 中的画图动作分到几个 fig 就可方便的实现动画效果。但实际 beamer 显示结果可能多个 fig 重叠而不是象在同一 fig 中时错落开，为此宜在每个 fig 中先定义整个大图的框（相当于画布）。象这样：
<src lang="metapost">
pickup pencircle scaled 0.0;
draw (-5.0u,-5.0u); draw (5.0u,1.0u);%外外框
pickup pencircle scaled .5bp; %内格子
</src>

3.2 重复利用刚画的图（平移等）

用 currentpicture 这个变量，如
<src lang="metapost">
draw currentpicture shifted (0,-2u);
</src>

3.3 累进画图(fig.2 包含 fig.1)

<src lang="metapost">
shipout currentpicture;
charcode:=2;
</src>

3.4 循环 label

<src lang="metapost">
extrasep:=-3pt;

for i=1 upto m-1:
label.bot(char (i+ ASCII "1"), (i*u, extrasep));
label.lft(char (i+ ASCII "1"), (0, i*u));
endfor;
</src>

3.5 draw 中循环画正多边形

<src lang="metapost">
n:=5;
draw for i=0 step 1 until n-1:
1cm*up rotated (i*360/n) –
endfor cycle;
</src>

3.6 一些语句

关于 quartercircle, halfcircle, buildcycle(), abs(), angle(), decimal(),
unitvector, subpath,intersecttimes 等等。
<src lang="metapost">
path d; d = quartercircle scaled 2r rotated -90 &
halfcircle scaled 2r;
path bluepart; bluepart = buildcycle(lambda.top, d, lambda.rt);
drawarrow z.pi rotated 8 .. z.pi rotated 80 ..
z.pi rotated 178;
draw fullcircle scaled 2abs(O-A) shifted O;
drawarrow (origin–A) rotated -1/3 angle(A);
label.llft(decimal(angle(C-A)-angle(B-A)), A);
B := A + 1cm*unitvector(direction 0 of p);
draw subpath(0,1) of p withcolor red;

path p, q;
numeric a,b;
(a,whatever) = p intersectiontimes q;
draw point a of p withpen pencircle scaled 4bp;

for i=0 step length(p)/n until length(p):
drawarrow (point i of p) –
1cm * unitvector(direction i of p)
shifted point i of p;
endfor;
</src>
