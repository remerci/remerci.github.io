1 补些新发现的功能和设置

1.1 markup 功能

  • nested list: M-enter, C-<, C->
  • footnote: C-c ! a
  • 开始空 6 格居中，空少于 6 格引用，>开头同 verse 标签，;开头注释
  • cite, verse, comment 等标签
  • <verbatim>*强调，_下划线，= verbatim</verbatim>

1.2 设置

  • 简化设置 style

    MuseConfig 中用 xhtml 做 style，用到大量图片美化，html-header-rooter 用 elisp 文件来

做，css 也在该文件中指定，并定义了一个 relative-path 的函数指定图片路径。这显得有些
复杂，我做了简化处理，用 html 做 style，header 和 rooter 的 html 文件直接指定，css 文件也
通过变量指定。

  • 修改 css 改善 src 标签显示效果

    用了简单的 default.css 文件，参考其它 css 文件改变 example 和 src 标签显示的边框和底色。

这样 src 标签的高亮不再难看。

  • 使用变量和相对路径

    这样可增强可移植性。

3 安装及配置

下载 emacs-muse 扩展文件后放到 emacs-lisp.

配置要点：

  • 定义 projects

<src lang="elisp">
(setq muse-project-alist
`(
("OS"
("~/muse/source/OS"
:default "index"
:force-publish ("WikiIndex"))
(:base "wiki-xhtml" :path "~/muse/publish/OS"))
("Soft"
("~/muse/source/Soft"
:default "index"
:force-publish ("WikiIndex"))
(:base "wiki-xhtml" :path "~/muse/publish/Soft"))
("Research" (,@(muse-project-alist-dirs "~/muse/source/Research/") ;源文件路径
:default "index" ;默认项目主页
:force-publish ("WikiIndex"))
,@(muse-project-alist-styles "~/muse/source/Research/" ;还是源文件路径
"~/muse/publish/Research" ;发布路径
"wiki-xhtml"))
))
</src>

  • 设置 stylesheet 及目录样式

<src lang="elisp">
(unless (assoc "my-blosxom" muse-publishing-styles)
(muse-derive-style "wiki-xhtml" "xhtml"
:header "~/muse/common/my-header.html"
:footer "~/muse/common/my-footer.html"
))

(custom-set-variables
'(muse-html-style-sheet "<link rel=\"stylesheet\" type=\"text/css\" charset=\"utf-8\" media=\"all\" href=\"../common/stylesheets/core.css\" />")
)
</src>

4 注意

4.1 与 emacs-wiki 区别：

  • 通过.muse 后缀自动加载 muse-mode（无后缀可能也行，还不会配置）
  • 列表的-和 1.前要加空格才会识别

4.2 源代码高亮

apt-get 安装 source-highlight，并在.emacs 中相应配置。使用时这样写：
<example>
<source lang="c">
</source>
</example>
支持很多语言和输出格式，如：
cpp.lang: C, H, c, cc, cpp, h, hh, hpp.

事实上 muse 现在已支持通过 htmlize 的语法高亮，使用与上同，只是 tag 为 src。
只要 emacs 支持 lang-mode，则可对相应的 lang 高亮，如下面的 nesc 高亮：

<src lang="nesc">
module muTeslaM {
provides {
interface StdControl;
interface muTesla;
}
uses {
interface StdControl as RadioControl;
interface StdControl as KeyChainControl;
interface StdControl as TimerControl;

interface SendMsg as SendParaMsg;
interface ReceiveMsg as ReceiveParaMsg;
interface SendMsg as SendKeyMsg;
interface ReceiveMsg as ReceiveKeyMsg;

interface Timer as TickTimer;
interface SysTime;
interface Leds;
interface Random;
interface SHA1; //used for one-way key chain
interface Primitive;
interface Buffer; //buffer the non-authenticated sync messages.
interface KeyChain; //one way key chain

//for debugging
#ifdef DEBUG_SERIAL
interface Serial; //used Cricket for debug
#endif
}
}
</src>

4.3 发布时出现 error "Invalid face"

从网上查到, 更改 htmlize.el 文件即可:
<example>
In Emacs 23.X,

(face-attribute 'default :inherit) => nil

while in Emacs 22.X,

(face-attribute 'default :inherit) => unspecified

I'm not sure whether that's a bug, but it is an incompatible change.

For the time being, you can patch your htmlize.el, changing one line
of htmlize-face-size, from

for f = face then (face-attribute f :inherit)

to

for f = face then (or (face-attribute f :inherit) 'unspecified)

Juanma
</example>
