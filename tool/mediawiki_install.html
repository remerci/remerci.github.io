1 安装 php 引擎

windows installer 安装包对于 IIS 来说只能以 CGI 方式安装，不能以 server module 方式安装。故采用 zip 手动安装方式。
下了最新版的 php zip 包，按照网上给出的方法一直装不好。
最后还是静下心来，按照里面带的 install.txt 文件一步一步来，终于搞定了。

  • 解压到 c:\php5，将其加到 PATH 环境变量里，并设为 PHPRC 环境变量的值。
  • 设置 c:\php5 对所有用户可读
  • 修改 php.ini-recommended 为 php.ini，修改下面的一些选项：
  • doc_root = "website 根目录"
  • extension_dir = "c:\php\ext"
  • 查找“extension”，把需要用到的扩展模块前的；号去掉。如需要用 mysql 的就是 php_mysql.dll
  • IIS 支持配置，网站属性对话框里
  • ISAPI Filters, add a new ISAPI: php5isapi.dll
  • Under 'Home Directory', 'Virtual Directory', or 'Directory', Change the Execute Permissions to 'Scripts only';
  • 点击“配置”按钮，在“映射”选项卡中，点“添加”按钮，通过“浏览”，找到 php 解压目录下的 php5isapi.dll 文件，确定，“扩展名”填“.php”，其他选项默认即可。
  • 找到 IIS 的 web 服务扩展，点击“添加一个新的 web 服务扩展”，扩展名可以为“php”，添加文件同样为 php5isapi.dll，并把“设置扩展状态为允许”勾选。
  • 在开始-运行里输入 iisreset，确定，重启 IIS。至此，在 IIS6 上安装 PHP5 已经完成。

2 安装 mysql 和 mediawiki

这里有一个比较常见的错误，就是不能“Start service”，一般出现在以前有安装 mysql 的服务器上，解决的办法，先保证以前安装的 mysql 服务器彻底卸载掉了；不行的话，检查是否按上面一步所说，之前的密码是否有修改，照上面的操作；如果依然不行，将 mysql 安装目录下的 data 文件夹备份，然后删除，在安装完成后，将安装生成的 data 文件夹删除，备份的 data 文件夹移回来，再重启 mysql 服务就可以了，这种情况下，可能需要将数据库检查一下，然后修复一次，防止数据出错。

不需要在 mysql 里直接增加用户。打开 mediawiki 网页后的配置网页通过 root 密码可以直接建立好数据库，用户等。

3 使用配置

默认情况下只允许上传图象文件，可以通过修改 2 个变量解决。

在/includes/DefaultSettings.php 文件中修改$wgVerifyMimeType 这个参数为 false，不对
上传的文件格式进行检查。$wgFileExtensions 参数改成下面这样：
<src lang="perl">
$wgFileExtensions = array(’png’, ‘gif’, ‘jpg’, ‘jpeg’, ‘doc’, ‘xls’, ‘pdf’);
</src>
